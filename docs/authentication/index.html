Introduction​

Nour-API provides a simple, token-based authentication system. Since Nour-API is stateless, it does not support session-based authentication like the full Laravel framework. Instead, authentication is typically handled using API tokens or headers.

Configuration​

Before using Nour-API's authentication features, you need to configure the authentication service provider. In your bootstrap/app.php file, you should uncomment the following line:

$app->register(App\Providers\AuthServiceProvider::class);

You should also uncomment the authentication middleware:

$app->routeMiddleware([
    'auth' => App\Http\Middleware\Authenticate::class,
]);

Authenticating Users​

Basic Authentication​

Nour-API includes a built-in authentication guard that will validate API tokens. To use this guard, you need to set the auth.defaults.guard configuration option to api in your config/auth.php file:

'defaults' => [
    'guard' => 'api',
    'passwords' => 'users',
],

'guards' => [
    'api' => [
        'driver' => 'token',
        'provider' => 'users',
    ],
],

Token Authentication​

The most common authentication method for APIs is token-based authentication. Here's how you can implement it:

  1. Add an API token column to your users table:
Schema::table('users', function ($table) {
    $table->string('api_token', 60)->unique()->nullable();
});
  1. Generate tokens for your users:
$user = User::find(1);
$user->api_token = str_random(60);
$user->save();
  1. Configure the token guard in your AuthServiceProvider:
public function boot()
{
    $this->app['auth']->viaRequest('api', function ($request) {
        if ($request->input('api_token')) {
            return User::where('api_token', $request->input('api_token'))->first();
        }
    });
}

Custom Authentication​

You can also implement custom authentication logic by defining your own authentication guard:

public function boot()
{
    $this->app['auth']->viaRequest('api', function ($request) {
        // Custom authentication logic
        $token = $request->header('Authorization');
        
        if ($token) {
            // Remove 'Bearer ' prefix if present
            $token = str_replace('Bearer ', '', $token);
            return User::where('api_token', $token)->first();
        }
    });
}

Protecting Routes​

Once you have configured authentication, you can protect your routes using the auth middleware:

$router->get('profile', ['middleware' => 'auth', function () {
    // Only authenticated users may enter...
}]);

You can also protect controller actions:

$router->get('profile', [
    'middleware' => 'auth',
    'uses' => 'UserController@showProfile'
]);

Accessing the Authenticated User​

Once a user is authenticated, you can access the authenticated user via the Auth facade:

use Illuminate\Support\Facades\Auth;

$user = Auth::user();

Alternatively, you can access the authenticated user via the request instance:

use Illuminate\Http\Request;

public function profile(Request $request)
{
    $user = $request->user();
}

Determining If The Current User Is Authenticated​

To determine if the user is already logged in, you may use the check method:

use Illuminate\Support\Facades\Auth;

if (Auth::check()) {
    // The user is logged in...
}
